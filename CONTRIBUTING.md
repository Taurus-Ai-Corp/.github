## Quick Start

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Sign the CLA (first-time contributors)
5. Submit a pull request

## CLA Requirement

All contributors must sign our [Contributor License Agreement](CLA.md).

**Why?** We use a Triple-Plus Licensing Model that requires clear IP rights.

**How?** Comment on your PR: `I have read the CLA Document and I hereby sign the CLA`

## Development Guidelines

- Follow existing code style
- Write tests for new features
- Update documentation as needed
- Keep commits atomic and descriptive

## Questions?

- GitHub Issues for bugs/features
- Email: dev@taurusai.io
- Discord: [Join our community](https://discord.gg/taurusai)
