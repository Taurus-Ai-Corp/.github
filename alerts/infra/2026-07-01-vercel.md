**Root cause:** `prisma generate` fails because `@prisma/client/runtime/query_engine_bg.postgresql.wasm-base64.js` is missing from `node_modules`. This typically happens when:
- The Prisma version in `package.json` was bumped without a corresponding `@prisma/client` update
- The monorepo workspace hoisting placed the Prisma runtime in the wrong location
- The `postinstall` script that runs `prisma generate` runs before the WASM file is available

**Fix:**
1. Ensure `prisma` and `@prisma/client` versions are in sync in `package.json`
2. Verify the `prisma generate` script runs after all packages are installed (check install order in monorepo)
3. Add `"postinstall": "prisma generate"` under the correct workspace package if missing
4. Try pinning `prisma` to the last known working version and redeploy
