Both failures stem from a single PR merge: `Taurus-Ai-Corp/GRIDERA` PR #20, "fix(guard): migrate @hashgraph/sdk → @hiero-ledger/sdk", merged to `main` (commit `c7bbcc63e7317c4ecd47f2548e1efae12406e272`).

- **comply** — `dpl_DPc9erwJifXgvmCdiGKvppdsVyLe`, 2026-06-29, **ERROR** (production target).
- **landing** — `dpl_G873YsWRSono1wH5cMn83DzKwrCo`, 2026-06-29 (same merge timestamp), **ERROR** (production target).

Build log (tail, identical for both):
```
Error: Node.js version 20.x is deprecated. Deployments created on or after 2026-10-01 will fail to build. Please set Node.js Version to 24.x in your Project Settings to use Node.js 24.
 ERR_PNPM_OUTDATED_LOCKFILE  Cannot install with "frozen-lockfile" because pnpm-lock.yaml is not up to date with <ROOT>/packages/guard/api/package.json

   Failure reason:
   specifiers in the lockfile ({"@fastify/cors":"^11.0.0","@taurus/db":"workspace:*","fastify":"^5.3.0","fastify-plugin":"^5.0.0","@hashgraph/sdk":"^2.81.0","@types/node":"^22.0.0","tsup":"^8.0.0","typescript":"^5.5.0","vitest":"^3.0.0"}) don't match specs in package.json ({"@types/node":"^22.0.0","@hiero-ledger/sdk":"^2.82.0","tsup":"^8.0.0","typescript":"^5.5.0","vitest":"^3.0.0","fastify":"^5.3.0","@fastify/cors":"^11.0.0","fastify-plugin":"^5.0.0","@taurus/db":"workspace:*"})
Error: Command "pnpm install" exited with 1
```

PR #20 updated `packages/guard/api/package.json` (removed `@hashgraph/sdk: ^2.81.0`, added `@hiero-ledger/sdk: ^2.82.0`) but did not regenerate/commit `pnpm-lock.yaml`. Vercel's build pipeline runs `pnpm install --frozen-lockfile`, which hard-fails the moment any workspace's specifiers drift from the lockfile — this breaks the entire monorepo install step for every app that depends on `packages/guard/api` (here, both `comply` and `landing`), even though only one workspace package changed.

No runtime logs (build-time/install failure, never reaches runtime).

## Recommended Fix

1. In the `GRIDERA` repo, on a branch off `main`, run `pnpm install` locally (regenerating `pnpm-lock.yaml` against the updated `packages/guard/api/package.json`), commit the updated lockfile, and push.
2. Redeploy `comply` and `landing` (or just merge the lockfile fix — Vercel will auto-redeploy on push to `main`).
3. Consider adding a CI check (`pnpm install --frozen-lockfile` in a lint/test job) that fails fast on lockfile drift *before* merge, so future dependency-only PRs can't reach `main` without a synced lockfile.

**Secondary, non-blocking:** Both build logs warn that Node.js 20.x is deprecated and deployments created on/after 2026-10-01 will fail to build. Recommend updating the Node.js Version to 24.x in Vercel Project Settings for `comply` and `landing` (and likely the rest of the fleet) ahead of that date.
