## Recommended Fix

### For `frontend` — prevent recurrence

1. **Framework config**: Ensure `vercel.json` explicitly sets `"framework": "vite"` (or the correct
   framework) so Vercel never falls back to Next.js auto-detection on this project.

2. **TypeScript strict mode guard**: The TS2393 (duplicate function) and TS1484 (`import type`)
   errors suggest generated/merged code was committed without a local `tsc --noEmit` check.
   Add a pre-push hook or CI step:
   ```bash
   npm run build   # or: tsc --noEmit
   ```
   This would have caught both failures before push.

3. **`verbatimModuleSyntax` imports**: Any type imported alongside value imports from the same
   module needs `import type { Foo }` when `verbatimModuleSyntax: true` is set in `tsconfig.json`.
   Quick fix pattern:
   ```ts
   // Before
   import { Campaign, someValue } from './types'
   // After
   import type { Campaign } from './types'
   import { someValue } from './types'
   ```
